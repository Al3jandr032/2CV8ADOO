Gestor de Citas Médicas


Equipo II

Objetivo general

Desarrollar un sistema de software que sea capaz de agendar de manera electrónica las citas de un usuario a una de las diferentes unidades médicas, considerando los horarios de la unidad médica, el horario del médico y el tipo de servicio, todo esto con el fin de poder tener un registro de los visitas de un paciente y lograr dar seguimiento a su caso, facilitando el manejo de los tiempos y documentación de los consultorios médicos.

  

Objetivos Específicos

Desarrollar una aplicación  que permita

  • Crear una cita desde cualquier lugar, para asistir a la unidad médica, horario, médico, y tipo de servicio de preferencia.


  • Una receta medica firmada electronicamente.

Problematica


Actualmente el IPN cuenta con unidades médicas distribuidas en diferentes ubicaciones, atendiendo a la comunidad politécnica, dentro de cierto horario, este proceso se realiza sin ninguna restricción.
 
Ademas dar seguimiento a un paciente con el sistema de citas actual podría surgir situaciones inconvenientes como:  pérdidas de documentos y listas, así como la posibilidad de traspapelar documentos de pacientes diferentes ocasionando que se lleve un seguimiento inadecuado del paciente.

Interfaz de Usuario

La interfaz gráfica utilizara menús desplegables para fácil selección de las opciones, además  Utilizará los colores oficiales del IPN.


Interfaces externas de Software y Hardware

El sistema se conectara con un módulo de software externo para la firma de la receta médica a través de un web servicie, también con otro encargado de gestionar el expediente médico electrónico 

Eficiencia


  • El sistema utilizara almacenamiento redundante para mantener los datos disponibles ante cualquier problema de hardware.

  • El respaldo de los datos almacenados en el servidor de base de datos.

  • El respaldo se realizara cada noche entre las 12 y 1 de la mañana.

Mantenimiento

  • Se deberá poder dar mantenimiento al front-end y back-end por separado sin tener que alterar al contrario cuando uno cambia

Portabilidad

·         El sistema podrá ser visualizado en los navegadores actuales, independientemente del sistema operativo utilizado, con una conexión mínima recomendada de 1Mbps


Restricciones de Diseño y Construcción


·  Se utilizara una arquitectura cliente-servidor en la aplicación web

·   Se utilizaran los medios de comunicación requeridos por los web service de los módulos externos al sistema
